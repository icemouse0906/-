# Ultimate Whisper 服务离线部署手册 (Windows ARM64)

## 一、准备工作

1.  确保服务器上已安装 Docker Desktop for Windows，并已启动。
2.  确保服务器上已安装 Python (建议 3.9 或更高版本)。
3.  将本光盘所有内容，完整拷贝到服务器的任意位置（例如桌面）。

---

## 二、一键部署服务

1.  进入拷贝好的文件夹。
2.  找到 `run.ps1` 文件。
3.  **右键点击 `run.ps1`，选择“使用 PowerShell 运行”**。 (如果菜单中有“以管理员身份运行”，请优先选择它)。
4.  在弹出的“用户账户控制(UAC)”窗口中，点击“是”。
5.  脚本会自动离线安装Python依赖包、加载Docker镜像并启动服务。整个过程需要几分钟，请根据提示耐心等待。
6.  看到“部署成功”的提示后，服务即在后台运行。

---

## 三、(可选) 手动功能验证

如果您想在服务器上亲自操作，验证服务的功能，我们推荐在一个独立的“虚拟环境”中进行，以保持系统干净。

**1. 打开 PowerShell 窗口：**
   - 在当前文件夹的地址栏输入 `powershell` 并按回车。

**2. 创建并激活虚拟环境：**
   - 在 PowerShell 窗口中，运行以下命令来创建虚拟环境 (此步骤只需执行一次)：
     python -m venv venv

   - 接着，运行以下命令来激活它 (每次打开新的 PowerShell 窗口测试时都需要激活)：
     .\venv\Scripts\activate

   - 成功后，命令行提示符前会出现 `(venv)` 字样。

**3. 运行测试脚本：**
   - 将一个音频或视频文件（如 aac, flac, mp3, mp4, wav 等）放到当前文件夹。
   - 在同一个 PowerShell 窗口中，运行以下命令 (将 "你的文件名.mp4" 替换成真实文件名):
     python transcribe_and_translate.py "你的文件名.mp4"

   - 脚本会开始处理文件，并最终打印出识别和翻译的结果。

**4. 退出虚拟环境：**
   - 测试完毕后，运行以下命令即可退出虚拟环境：
     deactivate

---

## 四、如何检查和停止服务

- **检查正在运行的服务**：在 PowerShell 中运行 `docker ps`
- **查看某个服务的日志**：在 PowerShell 中运行 `docker logs ultimate_whisper_service`
- **停止并移除所有服务**：在当前文件夹下的 PowerShell 中运行 `docker compose down`
